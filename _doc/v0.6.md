# git4 - 開發紀錄 v0.6

**日期**: 2026-05-01
**版本**: v0.6

## 開發摘要
在這個版本裡，我們正式讓 `git4` 踏出本機電腦，與真實的外部網際網路世界產生交集。
我們實作了 `remote` 的管理機制與基本組態檔持久化，並開發了 `ls-remote` 來實際向 GitHub 伺服器探測該儲存庫的狀態，為後續正式下載開源專案打下堅實的基礎。

## 實作細節

### 1. `remote` 資源管理 (`.git4/config`)
- 實作了 `git4 remote add <name> <url>`，其會將傳入的名字與網址，以簡易的 INI 格式儲存到 `.git4/config` 之中（例如 `[remote "origin"]` `url = ...`）。
- 同時撰寫了 `get_remote_url` 的剖析函式，日後所有需要取用 URL 的網路操作均能讀取這份設定檔，讓操作大大簡化。

### 2. 與 GitHub 伺服器進行 Smart HTTP 溝通 (`ls-remote`)
- 為了確保依賴簡潔乾淨並且無須陷入深度的 Async/Tokio 異步迴圈中，我們引進了極簡的同步 HTTP 庫 `ureq`。
- 開發了 `git4 ls-remote <remote>`:
  - 對儲存庫底下的 `info/refs?service=git-upload-pack` 發起 `GET` 請求。
  - **Pkt-Line Format 解析器**：GitHub 返回的資料並非一般的 JSON 或是純文字，而是特有的 Pkt-Line 封包。我們撰寫了一個專門切分位元組，能精準利用前 4 個 16進位長度字串來動態推進與剝離 `0000` (Flush Packet) 的解譯器。
  - 成功剖析出每條長度正確的資源點，並用空白/Null字元截斷印出了如 `<SHA-1> refs/heads/main` 等指標歷史。

### 3. 自動測試順利連網
- 更新了 `test.sh` 加入 `[11/11]` 階段測試。
- 指令綁定了真實存在的 `https://github.com/cccrust/git4.git` 並成功調用 `ls-remote`，畫面精準無誤地取出了 `main` 以及剛發布的各個資源 `tags (v0.3, v0.4...)`！

## 備註與下一步
- 這是 `git4` 真正擁有 HTTP 通訊的重要大躍進！
