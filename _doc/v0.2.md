# git4 - 開發紀錄 v0.2

**日期**: 2026-05-01
**版本**: v0.2

## 開發摘要
延續 v0.1 的底層架構，v0.2 成功設計並實作了最重要的版控核心功能：**分支 (Branch)** 與 **切換還原 (Checkout)** 機制，讓 `git4` 正式具備了多線開發、還原舊檔案狀態的能力。

## 實作細節

### 1. 分支管理 (`branch`)
- 實作了 `git4 branch` 來列出所有存在的分支（並使用 `*` 標記當前 `HEAD` 所在的分支）。
- 實作了 `git4 branch <name>` 來建立新分支。其原理為擷取當下的 Commit Hash，並寫入到對應的 `.git4/refs/heads/<name>` 中做為指標。

### 2. 時間切換與檔案還原 (`checkout`)
- 實作了 `git4 checkout <name>` 命令，執行流程如下：
  1. **物件解析 (Resolve)**：嘗試解析 `<name>` 是單純的分支名稱，還是實體的 Commit Hash (例如由 log 取得的長雜湊)。
  2. **提取樹層 (Tree)**：從指定的 Commit 物件中取得底層對應的 Tree Hash。
  3. **實體還原 (Restore)**：實作了 `restore_tree` 方法，這是一個遞迴邏輯：它會讀取目標 Tree 以及子 Tree，把裡面的 Blob 分層依照路徑寫入實體檔案系統，同時更利用 Unix API (`fs::set_permissions`) 原汁原味還原檔案的 UNIX 執行與讀寫權限。
  4. **對齊指標 (HEAD)**：切換完成後更新 `.git4/HEAD`。如指向分支會改寫為 `ref: refs/heads/<name>`；若是純 Hash 則建立類似 "detached HEAD" 的直接參照狀態。

### 3. 自動化測試腳本升級
- 在 `test.sh` 中新增了 `[7/7]` 的 `branch` 與 `checkout` 測試區塊。
- 腳本成功模擬了：建立 `new-feature` 分支 $\rightarrow$ 切換分支 $\rightarrow$ 修改與提交新內容 $\rightarrow$ 確實驗證新 log 排列 $\rightarrow$ 再度切換回 `main` 分支。
- 此端對端測試已經成功完全通過，驗證了功能運作如同預期！

## 備註與下一步 (未來看法)
- v0.2 版的 `checkout` 負責將目標 Tree 內的檔案精確寫入並覆寫掉舊檔案。但目前的實作略過了**清除在新分支原本不存在的舊追蹤檔案**這項操作，未來若要強化嚴格的狀態淨空，可以加入與 `index` 的刪除同步。
- 下個階段（例如 v0.3）可以專注於實作並加強暫存區 (`index`) 的正式解析結構、或是實作 `git4 status` 來展示工作目錄與暫存區之間的狀態差異。
