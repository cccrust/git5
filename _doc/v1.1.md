# git5 - 開發紀錄 v1.1

**日期**: 2026-05-04
**版本**: v1.1
**狀態**: 從 git4 升級至 git5

## 升級摘要

本版本將原本單一 `main.rs` 檔案的專案重構為模組化架構，使程式碼更易於維護、測試與擴展。

## 模組結構

```
src/
├── main.rs          # CLI 入口，使用 clap 解析命令列參數
├── lib.rs           #  library 入口，匯出所有模組
├── object.rs        # 物件操作（讀寫、雜湊、顯示）
├── tree.rs          # Tree 物件處理（建立、讀取、還原）
├── commit.rs        # Commit 操作（建立 HEAD 管理、父母關係）
├── index.rs         # 索引管理（add、讀取 index、工作目錄掃描）
└── commands.rs      # 所有 Git 命令的實作
```

## 各模組職責

### object.rs
- `git4_dir()`: 尋找 .git4 目錄
- `read_object()`: 讀取並解壓縮物件
- `write_object()`: 將物件寫入儲存庫
- `hash_object()`: 計算檔案的 SHA1 hash
- `cat_file()`: 顯示物件內容

### tree.rs
- `write_tree()`: 遞迴建立 tree 物件
- `restore_tree()`: 將 tree 還原至工作目錄
- `get_head_tree()`: 取得 HEAD 對應的 tree hash
- `read_tree_recursive()`: 遞迴讀取 tree 結構

### commit.rs
- `get_head()`: 取得目前 HEAD 的 commit hash
- `update_head()`: 更新 HEAD 指向
- `commit_tree()`: 建立 commit 物件
- `resolve_revision()`: 解析分支名稱或 hash
- `get_commit_parent()`: 取得 commit 的父 commit
- `is_ancestor()`: 判斷是否為祖先 commit

### index.rs
- `read_index()`: 讀取索引檔案
- `add_files()`: 將檔案加入索引
- `get_workspace_files()`: 掃描工作目錄中的檔案

### commands.rs
- `run()`: 命令分發器
- 所有 Git 命令的實作：init, hash-object, cat-file, write-tree, commit-tree, add, commit, log, branch, checkout, status, diff, merge, clone, push, fetch, remote, ls-remote, unpack-objects
