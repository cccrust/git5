# git5 - 開發紀錄 v1.7

**日期**: 2026-05-04
**版本**: v1.7.0
**狀態**: 完成

## 開發摘要

本版本實作 ls-files 和 rev-parse 指令，並修復 resolve_revision 對 HEAD 的處理。

## 實作細節

### 1. 新增 ls-files 指令

在 `src/main.rs` 新增：
```rust
LsFiles {
    #[arg(long = "cached")]
    cached: bool,
}
```

在 `src/commands.rs` 新增 `ls_files()` 函式：
- 支援 `git5 ls-files` - 列出 index 中的檔案

### 2. 新增 rev-parse 指令

在 `src/main.rs` 新增：
```rust
RevParse {
    #[arg(short)]
    short: bool,
    #[arg(default_value = "HEAD")]
    revision: String,
}
```

在 `src/commands.rs` 新增 `rev_parse()` 函式：
- 支援 `git5 rev-parse HEAD` - 取得 commit hash
- 支援 `git5 rev-parse --short HEAD` - 取得簡短 hash

### 3. 修復 resolve_revision

在 `src/commit.rs` 新增對 HEAD 的特別處理：
```rust
if name == "HEAD" {
    return get_head()?.ok_or_else(|| Git5Error::InvalidRef("HEAD has no commit".to_string()));
}
```

### 4. 新增整合測試

- `test_ls_files` - 測試 ls-files
- `test_rev_parse` - 測試 rev-parse
