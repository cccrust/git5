# git5 - 開發紀錄 v1.2

**日期**: 2026-05-04
**版本**: v1.2.0
**狀態**: 完成

## 開發摘要

本版本實現了錯誤處理的統一化，建立了 `Git5Error` 錯誤類型，取代原本散落的 `anyhow` 錯誤處理。

## 實作細節

### 1. 建立 error.rs 模組

建立 `src/error.rs`，包含：

- `Git5Error` enum 錯誤類型：
  - `NotARepository(String)` - 不是 git5 儲存庫
  - `ObjectNotFound(String)` - 物件不存在
  - `InvalidObject(String)` - 無效的物件格式
  - `InvalidRef(String)` - 無效的參照名稱
  - `Conflict(String)` - 合併衝突
  - `IoError(String)` - IO 錯誤包裝
  - `ParseError(String)` - 解析錯誤

- `Result<T>` 類型別名
- `From<std::io::Error>` 和 `From<hex::FromHexError>` 轉換實作

### 2. 更新各模組錯誤處理

所有模組的公開 API 現在使用統一的 `Result<T>` 類型：

- `src/lib.rs`: 匯出 `Git5Error` 和 `Result`
- `src/object.rs`: 使用 `Git5Error` 替換 `anyhow`
- `src/tree.rs`: 使用 `Git5Error` 替換 `anyhow`
- `src/commit.rs`: 使用 `Git5Error` 替換 `anyhow`
- `src/index.rs`: 使用 `Git5Error` 替換 `anyhow`
- `src/commands.rs`: 使用 `Git5Error` 替換所有 `anyhow!` 巨集呼叫

### 3. 新增測試

在 error.rs 中新增 3 個測試：
- `test_error_display` - 測試錯誤訊息顯示
- `test_error_debug` - 測試 Debug 格式
- `test_io_error_conversion` - 測試 IO 錯誤轉換
